## Why Cybersecurity

I've spent years writing software, but I kept running into the same gap in my knowledge: I could build systems, but I didn't fully understand how to break them. That matters. Writing secure code requires knowing what attackers actually do, not just following a checklist of best practices.

## The Program

I chose Northwestern's program because it runs six months of evening and weekend coursework built around labs, capture-the-flag exercises, and real tool usage — not lectures. It covers network security, penetration testing, SIEM tools, and defensive strategy, but what sold me was the format. I've done enough self-study to know that reading about Wireshark is not the same as using it to trace a suspicious packet across a network.

## What I'm After

My goal is specific: I want to combine software development with offensive security. Most developers treat security as an afterthought. Most security professionals don't write production code. I want to sit at that intersection — writing, say, a deployment pipeline that bakes in vulnerability scanning, or building internal tools that automate pen-test reporting.

I'll be documenting what I learn here, post by post, as I work through the program. Up first: security fundamentals, Linux command-line skills, and my first risk assessment exercise.
